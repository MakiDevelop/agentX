## 1. 專案目標總覽

### 原始目標
讓 agentX 成為一個真正「合格」的 Agent CLI（對標 Claude Code、Codex、Gemini CLI、Grok Build），特別是 headless 長任務的穩定性與可維護性。

### MT22 核心目標
解決長期存在的「雙任務系統分裂」問題：
- 舊系統：`task.py` + `TaskState`（單一任務）
- 新系統：`tasks.py` + `.agentx/tasks.json`（多任務清單，AgentSession 主力）

最終目標：讓新多任務系統成為**唯一真相來源**（Single Source of Truth），舊系統徹底退場。

## 4. 重要設計決策紀錄（MT22）

- **策略 B**：遷移成功後將舊 `task.json` 備份為 `task.json.bak.*`（而非直接刪除）。（使用者最終選擇）
- **Legacy 讀取統一**：所有讀取舊單一任務的行為，都必須經過 `_get_legacy_task_if_exists()`。
- **過渡期原則**：寧可多保留備份與相容，也不要讓使用者資料在過渡期間有風險。
- **可觀測性優先**：大量加入 warning、提示、診斷工具，讓過渡過程可被觀察（本次 `/doctor` 落地即為實踐）。
- **測試政策**：對 legacy 資料採「務實容忍 + 明確品質守衛」平衡（寬鬆日期結構、嚴格短 title/空 status 拒絕）。
